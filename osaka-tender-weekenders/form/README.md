# 予約フォーム（PHP）

入力（`../index.html` の RESERVATION）→ **確認画面** → **完了（サンキュー）画面** の3ステップ。
送信時に「管理者宛の予約通知メール」と「お客様宛の自動返信（サンキュー）メール」を送信します。

## フロー / ファイル
- `../index.html` … 入力フォーム（`action="form/confirm.php"` で POST）
- `confirm.php` … 入力検証 → 確認画面を表示（CSRFトークンを発行）
- `complete.php` … トークン検証 → メール2通送信 → サンキュー画面
- `lib.php` … 検証・CSRF・メール送信・共通レイアウト
- `smtp.php` … 依存ライブラリ不要のSMTPクライアント（STARTTLS/SSL/AUTH対応）
- `config.php` … 設定（送信先・送信元・件名・イベント情報・メール送信方式）

## 設定
運用に合わせて `config.php` を編集してください。
- `admin_email` … 予約通知の受信先（現状 `yoyaku260809@gmail.com`）
- `from_email` / `from_name` … 送信元。**できれば設置ドメインのアドレスに変更**（フリーアドレスを From にすると迷惑メール判定されやすい）
- 件名・イベント情報（自動返信本文に記載）

### メール送信方式（`mail_method`）
- `'smtp'`（現在の設定）… SMTPサーバー経由。送信元を独自ドメインにできSPFが通るため到達率が良い。
- `'mail'` … PHP の `mail()`（`mb_send_mail`）。

#### エックスサーバー（Xserver）でのSMTP設定 — 埋める値は3つ
`config.php` の以下を実際の値に置き換えてください（`★要変更` 箇所）:

1. `from_email` / `smtp.username` … 作成済みの**送信用メールアドレス**（例 `no-reply@ドメイン`）。両方とも同じ値。
2. `smtp.host` … 割当サーバー名 **`svXXXX.xserver.jp`**
   - 取得先: サーバーパネル →「サーバー情報」の**ホスト名**、またはメール「メールソフト設定」画面の**送信(SMTP)サーバー**。
3. `smtp.password` … その送信用メールアドレスのパスワード。

ポートは **465 + `'ssl'`（推奨）**。うまくいかない場合は **587 + `'tls'`（STARTTLS）** も可。

> 補足: このLPをエックスサーバー上に設置する場合、`mail_method` を `'mail'` にしてもドメインのSPFが効くため送信できます（最も手軽）。SMTPはより明示的に同じメールボックスから送る方式です。

送信失敗時は PHP の `error_log` に `[reserve][smtp]` で記録されます。

## サーバー要件
- **PHP 7.4+（推奨 8.x）** が動作すること
- **mbstring 拡張**
- メール送信方式に応じて:
  - `mail` … サーバーから `mail()` が送信できること（共有レンタルサーバーなら通常OK）
  - `smtp` … 対象SMTPへのアウトバウンド接続が可能なこと（587/465等）。`openssl` 拡張（TLS/SSL時）
- 到達率向上には、独自ドメインの From + SPF/DKIM の設定を推奨。

## セキュリティ / 仕様
- サーバー側で必須・形式（email / かな / tel / 枚数1〜4）を再検証
- CSRFワンタイムトークンで確認→完了を保護（リロードによる二重送信を防止）
- メールヘッダインジェクション対策（改行混入を拒否）
- ハニーポット（`website` フィールド）で簡易ボット対策
- 確認・完了ページは `noindex`

## 整理番号について
サンキュー画面・自動返信メールには「整理番号は追って担当者よりご連絡いたします」と記載。
管理者宛メールに予約内容が届くので、担当者が整理番号を割り振って個別に連絡する運用です。
